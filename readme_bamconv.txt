BAM Converter (bamconv)
~~~~~~~~~~~~~~~~~~~~~~~

About
~~~~~

BAM Converter is a supplementary tool that can be used to help generate configuration data from BAM input files. You
can specify one or more BAM input files. Additional BAM files will automatically be appended to the configuration in
the order of appearance. Cycle references are adjusted accordingly.

Configuration data can be generated in XML or JSON format. Optionally, the data can be printed to standard output, to
allow piping the content directly to the BAM Creator. A great number of available options allows you to customize the
generated configuration to your need.


Usage
~~~~~

Usage: bamconv [options] bamfile [bamfile2 ...]

Options:
--verbose                   Show additional log messages. All messages will be redirected to standard error if
                            configuration is written to standard output.
--silent                    Suppress any log messages. Useful when writing configuration data to standard output.
--log-style                 Print log messages in log style, complete with timestamp and log level.
--compact                   Generate compact configuration data (without indentations, line breaks, etc.).
                            Default: generate preformatted configuration data
--output-type type          Specify configuration type to output. Can be xml or json. Default: xml
--output configfile         Specify a filename where configuration data should be written to. By default configuration
                            data is written to standard output.
--bam-version version       Set BAM output version. Can be 1 for BAM V1 or 2 for BAM V2. Default: 1
--bam-output file           Set BAM output file.
--bam-pvrz-path path        Set PVRZ output path for BAM V2 output. Default: same folder as output BAM
--bam-search-path path      Search path for pvrz files associated with BAM V2 input files. Default: same path as input
                            BAM.
--bam-center                Specify to generate explicit center position entries for each individual BAM frame.
--bamv1-compress            Enable BAM V1 compression. Used by default.
--bamv1-no-compress         Disable BAM V1 compression.
--bamv1-rle type            Set RLE frame compression type. Allowed types:
                               -1     Decide based on resulting frame size
                                0     Always disable RLE encoding
                                1     Always enable RLE encoding
                            Default: -1
--bamv1-alpha               Preserve alpha in BAM V1 palette. Used by default.
--bamv1-no-alpha            Discard alpha in BAM V1 palette.
--bamv1-quality-min qmin    Set minimum quality for BAM V1 color quantization. Allowed range: [0, 100]. Default: 80
--bamv1-quality-max qmax    Set maximum quality for BAM V1 color quantization. Allowed range: [0, 100]. Default: 100
--bamv1-speed value         Set speed for palette generation. Default: 3
--bamv1-dither value        Set dither strength for output graphics. Value must be in range [0.0, 1.0]. Set to 0.0 to
                            disable. Default: 0.0
--bamv1-sort type           Sort palette by the specified type. The following types are recognized: none, lightness,
                            saturation, hue, red, green, blue, alpha. Append _reversed to reverse the sort order.
                            Default: none
--bamv1-palette file        Specify an external palette. Default: empty
--bamv2-start-index idx     Set start index for PVRZ files generated by BAM V2.
                            Allowed range: [0, 99999]. Default: 1000
--bamv2-encoding type       PVRZ pixel encoding type. Available types:
                                0     Determine automatically
                                1     Enforce DXT1 (BC1)
                                2     Enforce DXT3 (BC2) [unsupported by games]
                                3     Enforce DXT5 (BC3)
                            Default: 0
--bamv2-threshold value     Percentage threshold for determining PVRZ pixel encoding type. Allowed range: [0.0, 100.0].
                            Default: 5.0
--bamv2-quality value       Quality of PVRZ pixel encoding. in range [0, 2], where 0 is lowest quality and 2 is highest
                            quality. Default: 1
--bamv2-weight-alpha        Weight pixels by alpha. May improve visual quality for alpha-blended pixels. Used by default.
--bamv2-no-weight-alpha     Don't weight pixels by alpha.
--bamv2-use-metric          Apply perceptual metric to encoded pixels. May improve perceived quality.
--bamv2-no-use-metric       Don't apply perceptual metric to encoded pixels. Used by default.
--filter name[:key=value[;key=value[;more options...]]]
                            Define one or more filters complete with options. 'name' is the filter name, separated by
                            colon from zero, one or more options. Each option is defined by the option name 'key' and
                            the respective option value. Semicolon is used to separate multiple options. Wrap the whole
                            definition in quotes if it contains spaces. Add multiple --filter instances to define
                            multiple filters. Filters are added in order of appearance.
                            Example: --filter rotate:angle=12.5;interpolate=true;updatecenter=true
--help                      Print this help and terminate.
--version                   Print version information and terminate.
